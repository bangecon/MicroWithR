Overview

Basic Idea

Make a hypothesis of what would happen if one of the exogenous determinants changes.

  1. Ceteris paribus
  2. Small changes

Determinants

In the Cobb-Douglas example, the demand function for good \(i\) depends on:

  1. Preferences (the budget-share parameters, \(\alpha\) and \(1 - \alpha\));
  2. Income (\(m\));
  3. The price of good \(i\) (\(p_i\));
  4. The price(s) of the other good(s) (\(p_j\)).

We will focus mainly on income and price.

Types of Hypotheses

  1. Qualitative: Which direction?
  2. Quantitative: How much?

Approaches

  1. Graphical (as long as precise values aren’t important)
  2. Computational
  3. Anayltical

Price Change

Let’s analyze the effect of a $0.50 increase in price, and a $0.50 decrease in price for good 1.

  1. For a consumer with the same preferences (Cobb-Douglass with \(\alpha = 0.25\)), and facing the price of good 2 and income (\(p_2 = 1\)), compute the new equilibria for an increase in the price of good 1 from $1.50 to $2.00 (dp1 = 0.5); repeat for an equal decrease in \(p_1\) to $1.00.

In the Cobb-Douglas example, consumption of each good increases by equal proportions to the change in income (partly because of the fact that consumers spend a constant share of their budget on each good). This means that the income expansion path is a straight line from the origin.
(In fact, a linear income expansion path is a feature of many utility functions that have the property of being homothetic, which also includes the CES utility function.)

Analytical Approach

Basic approach:

  1. Find the demand functions in general form, without specific parameter values for \(m\), \(\alpha\), \(p_1\), and \(p_2\).
  2. Take the derivative of the demand functions with respect to each variable.

R can actually do this super-fast (by vectorizing the derivatives)!

Solving the first order conditions for \(x_1\) and \(x_2\) in terms of the parameters (the general-form solution from last time), we get the demand functions. Run the following code to get this result.

u <- ysym('x1^(alpha) * x2^(1-alpha)')
L <- u + ysym('lambda*(m - p1*x1 - p2*x2)')
dL <- deriv(L, c('x1', 'x2', 'lambda'))
MU <- deriv(u, c('x1', 'x2'))
MRS <- yac_str(MU[1]/MU[2]) |> 
  Simplify() |> 
  ysym()
D <- solve(c(MRS, dL[[3]]), c(ysym('p1/p2'), 0), c('x1', 'x2')) |> 
  y_rmvars() |>
  simplify()
D[1] <- with_value(D[1], 'x2', D[2]) |> 
  simplify()

\[x_1(p_1, m) = \frac{\alpha m}{p_{1}}\]

\[x_2(p_2, m) = \frac{\left( 1 - \alpha \right) m}{p_{2}}\]

Notice that for these preferences the increase in consumption for \(x_1\) is linear in the amount of income (it only depends on the exogenous parameters \(\alpha\) and \(p_1\)).

Run the following code to take the derivative of demand for \(x_1\) and \(x_2\) with respect to income and prices.

(statics <- deriv(D, c('m', 'p1', 'p2')))
{{           alpha/p1,        (1-alpha)/p2},
 {    (-alpha*m)/p1^2,                   0},
 {                  0, (-(1-alpha)*m)/p2^2}} 
(statics <- deriv(D, c('m', 'p1', 'p2')))

substituting our parameter values, we have:

\[\nabla x_1(m, p_1, p_2) = \begin{pmatrix} \frac{d}{dm}x_1(m, p_1) \\ \frac{d}{dp_1}x_1(m, p_1) \\ \frac{d}{dp_2}x_1(m, p_1) \end{pmatrix} = \begin{pmatrix} \frac{\alpha }{p_{1}} \\ \frac{ - \alpha m}{p_{1} ^{2}} \\ 0 \end {pmatrix} = \begin{pmatrix} \frac{0.25}{1.50} \\ \frac{-25}{1.50^2} \\ 0 \end{pmatrix} \approx \begin{pmatrix} 0.16667 \\ -11.11 \\ 0 \end{pmatrix}\]

and

\[\nabla x_2(m, p_1, p_2) = \begin{pmatrix} \frac{d}{dm}x_2(m, p_1) \\ \frac{d}{dp_1}x_2(m, p_1) \\ \frac{d}{dp_2}x_2(m, p_1) \end{pmatrix} = \begin{pmatrix} \frac{1 - \alpha }{p_{2}} \\ 0 \\ \frac{ - \left( 1 - \alpha \right) m}{p_{2} ^{2}} \end {pmatrix} = \begin{pmatrix} \frac{0.75}{1} \\ 0 \\ \frac{-75}{1^2} \end{pmatrix} \approx \begin{pmatrix} 0.75 \\ 0 \\ -75 \end{pmatrix}\]

Connecting Many Dots on a Demand Curve

The more points we do this for, the smoother and more complete our demand curve becomes. Run the code below to derive the demand schedule for \(x_1\) using the parameter values in our original example (m = 100, p2 = 1, \(\alpha\) = 0.25) for the prices ranging from 0.5 to 2.5 in increments of 0.1.

  1. Define p1seq as the sequence of prices and c1 as a NULL object.
  2. Create a for loop to iterate the constrained optimization function over all of the values in the length of p1 using the following steps:
  1. Defines the initial values of the choice variables, pars to be equal to c(1,1) if i == 1 and otherwise equal to the value of the solution from the previous iteration (which we call c below);
  2. Define c as the value of the pars result from the current iteration solution.
  3. Define the \(i^th\) element of c1 as the first element of the pars variable (the optimal level of consumption for that price) of each solution.
  1. Plot the demand curve of p1 (y-axis) versus c1 (x-axis).
p1seq <- seq(0.5, 2.5, 0.1)
c1 <- NULL
for (i in 1:length(p1seq)) {
  if(i == 1) {
    pars <- c(1,1)
  } else {
    pars <- c
  }
  c <- solnp(
    pars = pars,
    ux,
    ineqfun = function(x) p1seq[i] * x[1] + p2 * x[2],
    ineqLB = 0,
    ineqUB = m,
    LB = c(0, 0),
    UB = c(Inf, Inf)
  )$pars
  c1[i] <- c[1]
}

Income and Substitution Effects

The substitution effect arises because the change in the price of one good encourages a consumer to buy more of the relatively cheaper good (and less of the relatively more expensive good).
Analytically, the substitution effect is the difference between (1) the quantity that minimizes the consumer’s expenditures subject to leaving the the consumer at least as well-off as the original equilibrium (\(\tilde{x}\)); and (2) the original quantity consumed (\(x^*\)).

The income effect arises because when prices change, the same amount of money income can now buy a greater or lesser amount of goods and services, overall.
Analytically, the income effect is the difference between (1) \(\tilde{x}\); and (2) the new quantity consumed (\(x^**\))

Finding \(\tilde{x}\).

To separate the income and substitution, the first order of business is finding \(\tilde{x}\). Changes in Consumer Surplus

The two dashed expenditure lines (through \(x^**\) and \(\tilde{x}\)) capture the difference in income needed to achieve the new and original maximum levels of utility, respectively, calculated at the new price levels. In other words this is the additional income the consumer “feels” by moving from the original indifference curve to the new one. It is also one of the ways to capture the change in consumer surplus across all markets when relative prices change.

Calculating Consumer Surplus

Once you’ve found \(\tilde{x}\), finding the consumer surplus is relatively straightforward. In our case, the expenditures needed at the new prices in order to reach the original utility is the value of the expenditure function at \(\tilde{x}\), which equals the last value in xTilde$values, or 90.36; the expenditures needed to reach the new optimal utility is simply the consumer’s money income, m. The change in consumer surplus will be the difference 9.64.

Some policy analyses use this to evaluate the welfare effects of policy changes in terms of real income. The trick is estimating (with as few assumptions as possible!) the unobserved \(\tilde{x}\).
